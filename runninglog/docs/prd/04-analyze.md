## 1. 개요

### 1.1 목적

"기록 분석" 화면은 사용자의 월별 러닝 데이터를 시각적으로 요약하고, AI 기반 인사이트를 제공하여 러너의 성장과 훈련 방향을 안내하는 핵심 기능이다.

### 1.2 사용자 가치

| 가치 | 설명 |
|------|------|
| **데이터 시각화** | 6개월간의 추이를 차트로 한눈에 파악할 수 있어, 자신의 러닝 패턴과 성장 곡선을 직관적으로 이해할 수 있다 |
| **AI 코칭** | 단순 숫자 나열이 아닌, 각 지표별 AI 코멘트와 종합 분석을 통해 "지금 내가 어떤 상태인지", "다음 달에 무엇을 해야 하는지" 구체적인 방향을 받을 수 있다 |
| **동기부여** | 전월 대비 증감률, 친근한 비유(예: "치킨 두마리에 해당하는 칼로리"), 격려 메시지로 지속적인 러닝 동기를 부여한다 |
| **훈련 최적화** | 심박수 Zone 분포, 페이스 추이 등 고급 지표를 통해 훈련 강도와 밸런스를 점검할 수 있다 |

### 1.3 타겟 사용자

- 주 2회 이상 러닝을 기록하는 활성 사용자
- 자신의 성장을 수치로 확인하고 싶은 데이터 지향적 러너
- AI 코칭을 통해 훈련 방향을 잡고 싶은 중급 러너

## 2. 사용자 흐름

### 2.1 진입 경로

```
캘린더 탭 (기록 목록)
  └─ "분석보기" 버튼 탭
      └─ 기록 분석 화면 (전체 스크롤 페이지)
          ├─ 섹션 1: 누적 거리
          ├─ 섹션 2: 달리기 횟수
          ├─ 섹션 3: 누적 시간
          ├─ 섹션 4: 소비 칼로리
          ├─ 섹션 5: 페이스
          ├─ 섹션 6: 심박수 분포
          └─ 섹션 7: 종합 분석 + 피드백
```

### 2.2 주요 인터랙션 흐름

1. **진입**: 캘린더 탭에서 "분석보기" 버튼을 탭하여 기록 분석 화면으로 이동
2. **월 선택**: 헤더의 월 선택 드롭다운("2025년 1월 V")을 탭하여 분석 대상 월 변경
3. **스크롤 탐색**: 세로 스크롤로 각 분석 섹션을 순차적으로 탐색
4. **섹션 접기/펼치기**: 각 섹션의 접기/펼치기("-") 아이콘을 탭하여 관심 없는 섹션을 축소
5. **AI 종합 분석 확인**: 최하단의 "당신의 페이스메이커" AI 종합 분석 카드 확인
6. **피드백 제출**: "최고예요" 또는 "별로예요" 버튼으로 AI 분석 품질에 대한 피드백 제출

### 2.3 네비게이션

- **뒤로가기**: 헤더 좌측 뒤로가기 또는 스와이프 백 제스처로 캘린더 탭 복귀
- **월 변경**: 드롭다운 선택 시 해당 월 데이터로 전체 화면 갱신 (스크롤 위치 최상단 리셋)

|------|------|

| **차트 유형** | **세로 바 차트** (6개월) |

|  | - 과거 5개월: 회색(#D9D9D9) 바 |
|  | - 현재 월: 오렌지(#FF6B00) 바, 숫자 레이블도 오렌지 |
|  | - 데이터 없는 월: 바 높이 0, 레이블 "0" 표시 |
| **접기/펼치기** | 지원 ("-" 아이콘으로 접기, "+" 아이콘으로 펼치기) |

### 3.7 섹션 7: 종합 분석

#### 3.7.1 AI 페이스메이커 카드

| 항목 | 스펙 |
|------|------|
| **카드 스타일** | 연한 회색 배경의 라운드 카드, 패딩 충분히 확보 |
| **본문 내용** | AI가 생성한 종합 분석 텍스트. 아래 내용을 포함: |
|  | - 이번 달 훈련 성향 요약 (예: "이번 달은 '속도'보다 '지구력'에 집중하셨네요!") |
|  | - 핵심 수치 인사이트 (예: "누적 거리는 20% 늘었지만, 평균 페이스는 유지되었습니다") |
|  | - 심박수/체력 분석 (예: "특히 심박수 Zone 5 비중이 낮아진 것으로 보아, 몸에 무리 주지 않고 기초 체력을 탄탄히 다지는 아주 영리한 한 달을 보내셨습니다") |
|  | - 다음 달 추천 (예: "지구력이 충분히 올라왔으니, 다음 달에는 주 1회 정도 인터벌 훈련을 섞어 페이스 향상에 도전해 보는 건 어떨까요?") |
| **페이스메이커 라벨** | 카드 하단 우측, "당신의 페이스메이커" 텍스트 + 로봇 아이콘 (오렌지 원형 배경) |
| **텍스트 스타일** | 본문: 일반 폰트, 강조 키워드는 **Bold** 처리 (예: '속도', '지구력', 'Zone 5', '20%') |

#### 3.7.2 피드백 섹션

| 항목 | 스펙 |
|------|------|
| **안내 문구** | "더 좋은 기록 분석을 위해 피드백을 남겨주세요!" (중앙 정렬) |
| **피드백 버튼** | 2개의 아웃라인 버튼, 가로 나란히 배치 |
|  | - "최고예요" 버튼: 좌측, 엄지척(👍) 아이콘 + 텍스트 |
|  | - "별로예요" 버튼: 우측, 엄지아래(👎) 아이콘 + 텍스트 |
| **버튼 스타일** | 기본: 회색 아웃라인, 라운드 모서리 |
|  | 선택 시: 오렌지 아웃라인 + 오렌지 텍스트로 변경 (선택 상태 표시) |
| **선택 동작** | 하나의 버튼만 선택 가능 (토글), 선택 즉시 서버에 피드백 전송 |
| **재선택** | 같은 버튼 재탭 시 선택 해제, 다른 버튼 탭 시 선택 변경 |

## 4. 데이터 모델

### 4.1 월별 집계 데이터 (MonthlyAnalysis)

```typescript
interface MonthlyAnalysis {
  // 기본 정보
  userId: string;
  year: number;
  month: number;            // 1~12

  // 누적 거리
  totalDistance: number;     // km 단위
  distanceChangeRate: number; // 전월 대비 증감률 (%, +/-), 전월 데이터 없을 시 null

  // 달리기 횟수
  totalRunCount: number;     // 회
  runCountChangeRate: number | null;

  // 누적 시간
  totalDuration: number;     // 초 단위
  durationChangeRate: number | null;

  // 소비 칼로리
  totalCalories: number;     // kcal

  // 페이스
  avgPace: number;           // 초/km 단위 (예: 342 = 5'42")
  minPace: number;           // 최고 페이스 (가장 빠른, 초/km)
  maxPace: number;           // 최저 페이스 (가장 느린, 초/km)
  paceChangeRate: number | null;

  // 심박수 Zone 분포
  heartRateZones: {
    zone1Ratio: number;      // 0~100 (%)
    zone2Ratio: number;
    zone3Ratio: number;
    zone4Ratio: number;
    zone5Ratio: number;
  };

  // 차트용 6개월 히스토리
  recentMonths: MonthlyChartData[];
}
```

### 4.2 차트 데이터 (MonthlyChartData)

```typescript
interface MonthlyChartData {
  year: number;
  month: number;
  distance: number;          // km
  runCount: number;          // 회
  duration: number;          // 초
  calories: number;          // kcal
  avgPace: number | null;    // 초/km, 기록 없으면 null
  minPace: number | null;
  maxPace: number | null;
  heartRateZones: {
    zone1Ratio: number;
    zone2Ratio: number;
    zone3Ratio: number;
    zone4Ratio: number;
    zone5Ratio: number;
  } | null;
}
```

### 4.3 AI 분석 데이터 (AIAnalysis)

```typescript
interface AIAnalysis {
  id: string;
  userId: string;
  year: number;
  month: number;

  // 섹션별 AI 코멘트
  distanceComment: string;
  runCountComment: string;
  durationComment: string;
  caloriesComment: string;
  paceComment: string;
  heartRateComment: string;

  // 종합 분석
  overallAnalysis: string;   // 마크다운 또는 리치텍스트 (Bold 키워드 포함)

  // 메타
  generatedAt: string;       // ISO 8601
  modelVersion: string;      // AI 모델 버전
}
```

## 5. API 요구사항

### 5.1 월별 분석 데이터 조회

```
GET /api/v1/analysis/monthly?year={year}&month={month}
```

| 항목 | 스펙 |
|------|------|
| **인증** | Bearer Token (필수) |
| **파라미터** | `year` (number, 필수), `month` (number, 1~12, 필수) |
| **응답 200** | `MonthlyAnalysis` 객체 (6개월 히스토리 포함) |
| **응답 404** | 해당 월에 러닝 기록이 없는 경우 |
| **캐싱** | 해당 월이 이미 지난 경우 장기 캐싱 가능 (월 데이터는 변하지 않음). 현재 월은 단기 캐싱 (새 기록 추가 가능) |

### 5.2 AI 분석 생성/조회

```
GET /api/v1/analysis/ai?year={year}&month={month}
```

| 항목 | 스펙 |
|------|------|
| **인증** | Bearer Token (필수) |
| **파라미터** | `year` (number, 필수), `month` (number, 1~12, 필수) |
| **응답 200** | `AIAnalysis` 객체 |
| **응답 202** | AI 분석 생성 중 (아직 준비되지 않음). 클라이언트는 폴링 또는 재요청 필요 |
| **응답 404** | 해당 월에 분석 데이터 없음 (러닝 기록 부족 등) |
| **비고** | 최초 조회 시 AI 분석이 없으면 서버에서 자동 생성 트리거. 이후 조회 시 캐싱된 결과 반환. 월이 종료된 후 데이터가 변경되지 않으므로 한 번 생성된 분석은 재생성하지 않음 |

### 5.3 분석 가능 월 목록 조회

```
GET /api/v1/analysis/available-months
```

| 항목 | 스펙 |
|------|------|
| **인증** | Bearer Token (필수) |
| **응답 200** | `{ months: { year: number, month: number }[] }` - 분석 가능한 월 목록 (내림차순) |
| **용도** | 헤더 월 선택 드롭다운에서 선택 가능한 월을 결정하는 데 사용 |

### 5.4 피드백 제출

```
POST /api/v1/analysis/feedback
```

| 항목 | 스펙 |
|------|------|
| **인증** | Bearer Token (필수) |
| **요청 Body** | `{ analysisId: string, year: number, month: number, rating: 'positive' \| 'negative' }` |
| **응답 200** | `{ success: true }` |
| **응답 409** | 이미 피드백이 존재하는 경우 (업데이트로 처리) |
| **비고** | 동일 월 피드백 재전송 시 기존 피드백을 업데이트 (UPSERT) |

### 5.5 피드백 삭제 (선택 해제)

```
DELETE /api/v1/analysis/feedback?year={year}&month={month}
```

| 항목 | 스펙 |
|------|------|
| **인증** | Bearer Token (필수) |
| **응답 200** | `{ success: true }` |
| **용도** | 이미 선택한 피드백 버튼을 재탭하여 선택 해제할 때 사용 |

## 6. 상태 관리

### 6.1 화면 상태

```typescript
interface AnalyzeScreenState {
  // 선택된 월
  selectedYear: number;
  selectedMonth: number;

  // 분석 가능 월 목록
  availableMonths: { year: number; month: number }[];

  // 월별 분석 데이터
  monthlyData: MonthlyAnalysis | null;
  monthlyDataLoading: boolean;
  monthlyDataError: Error | null;

  // AI 분석 데이터
  aiAnalysis: AIAnalysis | null;
  aiAnalysisLoading: boolean;
  aiAnalysisError: Error | null;

  // 섹션 접기/펼치기 상태
  sectionCollapsed: {
    distance: boolean;       // 기본: false (펼침)
    runCount: boolean;       // 기본: false
    duration: boolean;       // 기본: false
    calories: boolean;       // 기본: false
    pace: boolean;           // 기본: false
    heartRate: boolean;      // 기본: false
    overall: boolean;        // 기본: false
  };

  // 피드백 상태
  feedbackRating: 'positive' | 'negative' | null;
  feedbackSubmitting: boolean;
}
```

### 6.2 상태 전이 규칙

| 액션 | 상태 변경 |
|------|-----------|
| **화면 진입** | `selectedYear`/`selectedMonth`를 현재 월로 초기화, `availableMonths` 조회, 해당 월 데이터 + AI 분석 동시 로딩 |
| **월 변경** | `selectedYear`/`selectedMonth` 업데이트, 해당 월 데이터 재조회, 스크롤 위치 최상단 리셋, 접기/펼치기 상태 초기화(모두 펼침) |
| **섹션 접기/펼치기** | `sectionCollapsed[section]` 토글. 애니메이션과 함께 콘텐츠 높이 변경 |
| **피드백 선택** | `feedbackRating` 업데이트 + 서버 전송. 같은 버튼 재선택 시 `null`로 초기화 + 서버 삭제 |
| **데이터 로딩 실패** | `monthlyDataError` 또는 `aiAnalysisError` 설정, 재시도 UI 표시 |

### 6.3 캐싱 전략

- 이미 조회한 월의 데이터는 로컬 캐시에 저장하여 월 변경 시 즉시 표시
- 현재 월 데이터: TTL 5분 (새 러닝 기록 추가 가능)
- 과거 월 데이터: TTL 24시간 (데이터 변경 없음)
- AI 분석 결과: 영구 캐시 (한 번 생성되면 변경 없음)

## 7. 엣지 케이스

### 7.1 데이터 없는 월

| 시나리오 | 처리 |
|----------|------|
| **선택한 월에 러닝 기록이 0건** | 빈 상태 일러스트 + "이 달에는 러닝 기록이 없어요.\n달려볼까요?" 메시지 + "러닝 시작하기" 버튼 표시 |
| **6개월 히스토리 중 일부 월에 기록 없음** | 해당 월 차트 바 높이 0, 레이블 "0" 표시 (차트 자체는 정상 렌더링) |
| **심박수 데이터 없음** (심박수 센서 미연동) | 심박수 분포 섹션 숨김 또는 "심박수 데이터가 없어요. 워치나 밴드를 연결해보세요." 안내 표시 |
| **칼로리 데이터 없음** (체중 미입력 등) | 추정 불가 시 "칼로리 데이터를 계산하려면 프로필에서 체중을 입력해주세요." 안내 |

### 7.2 AI 분석 로딩/에러

| 시나리오 | 처리 |
|----------|------|
| **AI 분석 생성 중 (202 응답)** | 각 섹션의 AI 코멘트 영역에 스켈레톤 로더(shimmer) 표시 + "AI가 분석 중이에요..." 메시지. 10초 간격 폴링 (최대 3회) |
| **AI 분석 생성 실패** | "AI 분석을 불러오지 못했어요." + "다시 시도" 버튼. 차트 데이터는 정상 표시 (AI 코멘트만 빈 상태) |
| **AI 분석 API 타임아웃** | 30초 타임아웃 후 에러 처리. 차트 데이터는 별도 API이므로 독립적으로 표시 |
| **AI 코멘트가 너무 긴 경우** | 각 섹션별 코멘트는 최대 2줄, 종합 분석은 제한 없음 (스크롤로 확인) |

### 7.3 첫 사용자 (온보딩)

| 시나리오 | 처리 |
|----------|------|
| **러닝 기록이 1건도 없는 사용자** | "분석보기" 버튼 비활성화 또는, 진입 시 온보딩 안내: "첫 러닝 기록을 만들어보세요! 기록이 쌓이면 AI가 분석해드려요." |
| **러닝 기록이 1건만 있는 경우** | 전월 대비 증감률 표시 불가 → 증감 배지 숨김. AI 코멘트는 단일 기록 기반으로 생성 (예: "첫 러닝을 기록하셨네요!") |
| **러닝 기록이 1개월분만 있는 경우** | 6개월 차트에서 해당 월만 표시, 나머지 월은 0으로 표시. 전월 대비 증감 불가 → 배지 숨김 |

### 7.4 기타 엣지 케이스

| 시나리오 | 처리 |
|----------|------|
| **네트워크 오프라인** | 캐싱된 데이터가 있으면 캐시 표시 + "오프라인 상태입니다" 배너. 캐시 없으면 "인터넷 연결을 확인해주세요" 에러 화면 |
| **월 변경 중 빠른 연속 탭** | 디바운스 적용 (300ms). 마지막 선택된 월만 API 호출 |
| **피드백 전송 실패** | 로컬에 피드백 상태 저장 후 다음 접속 시 재전송 시도. 사용자에게는 선택 상태 즉시 반영 (Optimistic UI) |
| **매우 긴 러닝 기록** (울트라마라톤 등) | 차트 Y축 스케일 자동 조정. 극단적 값이 있어도 다른 월의 바가 너무 작아지지 않도록 최소 높이 보장 |
